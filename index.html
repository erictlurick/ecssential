Installation

Version pré compilée

Une version standard et une version minifiée du fichier css sont disponibles à votre convenance dans le répertoire css/ prêts à être insérés dans votre fichier html.

ecssential/
├── css/
│   ├── styles.css
│   └── styles.min.css

À vous de compléter ou de modifier les styles prédéfinis en ajoutant vos propres déclarations css dans un nouveau fichier nommé, par exemple, theme.css et de l’appeler à la suite.

<html>
  <head>
    ...
    <link rel="stylesheet" href="css/styles.min.css">
    <link rel="stylesheet" href="css/theme.css">
    ...
  </head>
  ...
</html>

Version sass

Cette option permet un paramétrage plus poussé en donnant la possibilité d’activer ou désactiver certaines fonctionalités dans le fichier configuration.scss mais également de modifier les variables prédéfinies dans les fichiers global.scss et colors.scss situés dans le répertoire settings/. Il faudra ensuite compiler le fichier styles.scss pour rafraîchir vos déclarations de style.

Texte

Chapeau

Le chapeau (souvent écrit chapô est un texte généralement court, précédant le corps d’un article, et dont le but est d’en encourager la lecture en résumant le propos qui va être développé.

<p class="u-lede">…</p>

Paragraphes

Une suite de 45 à 75 caractères est communément considérée comme suffisante pour constituer une longueur de ligne de texte. La ligne de 66 caractères (en comptant à la fois les lettres et les espaces) représente un idéal souvent utilisé qui est possible avec les classes css .l-wrapper et .l-wrapper--narrow combinées.

Les lettrines sont composées d’une lettre initiale majuscule décorée placée en tête d’un texte et occupant une hauteur supérieure à la ligne courante. Les autres lettres du premier mot sont généralement en petites capitales. Comme lettre ornée, elle commence et décore une inscription, un paragraphe ou un chapitre d’ouvrage. Les moines du Moyen Âge cultivaient l’art de la lettrine dans leurs enluminures. Avec l’invention de l'imprimerie, la lettrine est devenue un ornement typographique. Elle peut provenir d’une vignette conçue à cet effet ou d’un caractère plus grand de la même police d’écriture ou d’une autre sorte.

<p class="u-dropcap">…</p>

Pour une présentation des paragraphes en alinéa activez là avec 'indent': true dans la liste de variables $typography qui se trouve dans le fichier scss/settings/configuration.scss

Marquage sémantique

  • L’element <a> est utilisé pour les hyperliens,
  • L’élément <abbr> est utlisé pour les abbréviations et les acronymes sont attribut title ne doit être utilisé que pour l’extension de l’abbréviation. Example : abbr,
  • L’élément <b> est utilisé dans une partie de texte en gras sémantiquement faible,
  • L’élément <strong> est utilisé pour donner de l’importance au texte,
  • L’élément <i> est utilisé pour différencier une partie du texte,
  • L’élément <cite> est utilisé pour citer le titre d’une oeuvre, d’un document ou d’un évènement,
  • L’élément <code> est utlisé pour déclarer du code informatique,
  • L’élément <del> est utilisé pour indiquer qu’une partie de texte est supprimée,
  • L’élément <dfn> est utilisé dans un paragraphe ou une liste de définitions pour représenter le terme d’une définition,
  • L’élément <em> est utilisé pour mettre une partie du texte en emphase,
  • L’élément <ins> est utilisé pour insérer un nouveau texte dans un document révisé,
  • L’élément <kbd> est utilisé lorsque le texte est utilisé pour définir des actions au clavier,
  • L’élément <mark> est utilisé pour marquer du texte, par exemple surligner un résultat de recherche,
  • L’élément <q> est utilisé pour une citation courte provenant d’une ressource externe,
  • L’élément <s> est utilisé pour identifier une partie de texte qui n’est pas correct ou non pertinent,
  • L’élément <samp> est utilisé dans un paragraphe pour écrire un échantillon de code,
  • L’élément <small> est utilisée comme contenu relatif mais non essentiel,
  • L’élément <span> est utilisé pour mettre en style une portion de texte qui se différencie des autres,
  • L’élément <sub> est utlisé pour déclarer un indice,
  • L’élément <sup> est utlisé pour déclarer un exposant,
  • L’élément <var> est utilisé pour déclarer une variable.

Espaces

Toute combinaison d’espace, de rentrée d’alinéa ou de retour à la ligne, est automatiquement comprise comme une espace mot. Si vous devez insérer plus qu’un espace mot entre deux phrases, ou tout autre caractère, utilisez alors l’un des caractères d’espace disponible en Unicode. Même si le caractère lui-même n’est pas présent dans le fichier de la police de caratères utilisée, les navigateurs qui comprennent l’Unicode pourront en afficher une bonne approximation. Résistez à la tentation d’utiliser l’espace insécable &nbsp; qui n’a de signification que pour lui-même.

  • Espace d’un em ou d’un cadratin avec &#8195;  
  • Espace numérique (égale à la chasse d’un zéro) avec &#8199;  
  • Espace d’un tiers de cadratin avec &#8196;  
  • Espace d’un quart de cadratin avec &#8197;  
  • Espace ponctuation (égale à la chasse d’une virgule) avec &#8200;  
  • Espace fine (1/5 cadratin, 1/8 en français) avec &#8201;  
  • Espace d’un sixième de cadratin avec&#8198;  
  • Espace ultra fine (de 1/10 à 1/16 de cadratin) avec &#8202;

Code

  • La balise <code> représente un bloc de code,
  • La balise <samp> indique que l’ordinateur produit un texte,
  • La balise <kbd> représente une entrée saisie au clavier,

Vidéos

Affichage de vidéos en iframe par défaut :

<figure>
  <iframe width="853" height="480" src="https://www.youtube.com/embed/8FqZZrbnwkM?rel=0&showinfo=0" frameborder="0" allowfullscreen></iframe>
</figure>

Modificateurs

Conserver le ratio de vidéos adaptatives en iframe avec la classe css .o-ratio :

<figure class="[ o-ratio  o-ratio--16:9 ]">
  <iframe width="853" height="480" src="https://www.youtube.com/embed/8FqZZrbnwkM?rel=0&showinfo=0" frameborder="0" allowfullscreen></iframe>
</figure>

Par défaut le ratio est de 1:1 avec la classe css o-ratio. La série de classes additionelles .o-ratio--2:1, .o-ratio--4:3 et o-ratio--16:9 permettent de le modifier. Activez ou désactivez ces modificateurs avec 'ratios': true ou 'ratios': false dans la liste de variables $objects qui se trouve dans le fichier scss/settings/configuration.scss

Conteneurs

  • La classe l-wrapper est utilisée pour définir un cadre fluide avec marges internes horizontales.
  • La classe modificatrice l-wrapper--wide est utilisée pour définir la largeur maximale du cadre à 1193px pour une taille de caractère de base de 16px.
  • la classe modificatrice l-wrapper--medium est utilisée pour définir la largeur maximale du cadre à 895px pour une taille de caractère de base de 16px.
  • La classe modificatrice l-wrapper--narrow est utilisée pour définir la largeur maximale du conteneur à 528px pour une taille de caractère de base de 16px.

Les valeurs de largeur maximale sont définies d’après l’échelle typographique en fonction de la taille de caractère et de l’interligne par défaut, voir modularscale.com.

Icônographie

SVG est un format d’image vectoriel reposant sur une syntaxe XML qui lui confère l’avantage d’être redimensionable sans perte de qualité. Avec Html 5 l’élément <svg> permet de définir un ensemble de pictogrammes directement dans une page. Il existe des outils permettant d’automatiser le processus dont Icomoon. Pour en savoir plus, voir cet article qui explique comment travailler avec des icônes en SVG.

Note : il est conseillé d’utiliser aria-hidden="true" pour garantir que les lecteurs d’écran ne liront pas l’icône à haute voix. Pour un lien ou un bouton dont le seul contenu est une icône il est recommandé de spécifier un texte accessible avec aria-label sur l’élément <a> ou <button>. Les définitions graphiques SVG font parti du document et ne seront pas mises en cache par le navigateur comme un sujet distinct ce qui signifie qu’elles ne seront pas réutilisable à travers les pages mais il est possible de stocker les données dans le navigateur de l’utilisateur avec javascript. La limite de stockage est habituellement de 5 Mo par domaine.
